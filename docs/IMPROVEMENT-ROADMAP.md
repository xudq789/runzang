# 润藏类命理站点：改进路线图与资料摘要

> 面向：**静态页 + AI 接口 + 支付** 的娱乐/工具型落地站。  
> 执行策略：能直接改代码的已做；需你授权、商户后台或真机环境的记在「搁置项」。

## 一、外部资料里的共性结论（精简）

| 方向 | 做法 | 来源类型 |
|------|------|----------|
| 转化 | 首屏说清「给谁、得到什么」；主 CTA 在移动端拇指区；减少干扰 | 落地页转化类文章 |
| 性能 | LCP &lt; 2.5s 量级目标；大图懒加载、关键图优先、预连接 CDN | Core Web Vitals / 性能博客 |
| 无障碍 | 跳过导航、`main` 地标、`focus-visible`、尊重 `prefers-reduced-motion` | WCAG 实践 |
| 信任 | 免责声明、常见问题、支付方式说明，降低付费疑虑 | 通用 UX |

---

## 二、已在本仓库落地的实现（本轮）

- **SEO / 分享**：规范链接、`meta` 补充、Open Graph / Twitter（占位图可用 COS 图，后续可换专用 1200×630）。
- **性能**：COS 域名 `preconnect`；首屏图 `fetchpriority`/`decoding`；明细图 `loading="lazy"`。
- **无障碍**：跳过链接、`<main id="main-content">`、标题用 `.visually-hidden` 替代 `display:none`。
- **体验**：`prefers-reduced-motion` 减弱滚动动画；键盘 `:focus-visible`。
- **创意模块**：「今日锦囊」按日期轮换短句；移动端 **底部 sticky「开始测算」**；**常见问题**折叠区。
- **其它**：`manifest.json`、`robots.txt`；打印样式（报告区更利于打印）。
- **文案**：「我已支付」确认框改为同时提及支付宝与微信。

详见代码：`index.html`、`css/styles.css`、`js/core/ui-enhancements.js`、`js/core/main.js`。

## 三、建议下一波（未实现 / 部分依赖你）

1. **Lighthouse / 真实性能分数**：需在本地或 CI 跑审计；若使用 Cursor 内置浏览器或联网 Lighthouse，**请先在 Cursor 里开启对应权限/功能**，再让助手跑分并逐项消红。
2. **PWA 图标**：仓库内暂无 `favicon`/多尺寸 icon；manifest 已可装，补 `icons` 后添加到主屏幕更完整。
3. **结构化数据**：可添加 `WebApplication` 或 `FAQPage` 的 JSON-LD（注意合规表述，避免医疗/迷信承诺）。
4. **A/B 测试**：标题、CTA 文案、解锁区价格展示（需统计后台或第三方）。
5. **服务端**：速率限制、缓存、CDN 缓存策略——在 `payment-backend`，非本静态仓库。
